Lesson 1 - Starting with p5.js

Welcome to the first “real” lesson of the workshop! In this lesson we will explore the p5.js official documentation, learn fundamentals for programming, and begin our first programming exercise.

p5.js Official “Get Started”

Popular open source tools often have an impressive team of contributors dedicated to creating wonderful official documentation. p5.js is a perfect example of this. Instead of reinventing the wheel, this workshop will point to official documentation as much as possible. Reading and understanding documentation is an important skill that when practiced will open the door to learning more and more technologies.

Let us start by using the p5.js official Get Started documentation to get the hang of the p5.js web editor. Following along until reaching the “What Next?” section should take less than 5 minutes. This page is not necessarily targeted for beginners, so do not feel bad if you do not understand the examples you are copying. Once you are finished, continue with Default Code Analysis to analyze the examples we ran in more detail.

Default Code Analysis

First off, congratulations if this was your first time reading official programming documentation! Throughout the course you will be doing much more of this. You now have knowledge about the basics of the p5.js web editor, how to input and change code, and how to run that code. So now, let us analyze the default code that exists when you open the web editor:

function setup() {
    createCanvas(400, 400);
  }
  
  function draw() {
    background(220);
  }

This code is defining two functions, setup and draw. These are important functions in p5.js because they are called at specific times. When you press play, setup is called once and then draw is called for the refresh rate of your display per second (60 times per second for most monitors). The contents of each function are included between { and }.

So now let us look at the contents of the setup function:

createCanvas(400, 400);

As you may have guessed, this creates a canvas, the area for graphics, with a 400 pixel width and 400 pixel height. Using programming vocabulary, this can be described as calling the function createCanvas with the the parameters or arguments 400 and 400. p5.js gives you many functions like this to do things like drawing shapes, handling input, playing sounds, and more.

The p5.js reference page lists all these functions, explains the parameters they take, describes what they do, and often contain an example using that function. Instead of focusing on memorizing the all of these functions, it is much more useful to learn how to efficiently use the reference page. Check out the createCanvas() reference to see how a function you already know is explained.

Errors

Understanding errors is a hugely useful skill for programming. When doing your exercise 1, it is unlikely you will write perfect error-free code your first time. It is very easy to make some mistake or typo while coding, but error messages make finding and fixing mistakes easy. While the “Get Started” page briefly covered errors, I would like to look at a specific example. Let us use this very simple code:

function setup() {
    createCanvas(400, 400);
    circle(40,40);
  }

The code is simply aiming to create a canvas, and draw a circle at (40, 40). It does not even define an update function, which is perfectly legal. However, when running this code in the editor, we see an error in the console.

🌸 p5.js says: circle() was expecting at least 3 arguments, but received only 2. (on line 73 in about:srcdoc [about:srcdoc:73:3]). (http://p5js.org/reference/#/p5/circle)

The most important part of this error is circle() was expecting at least 3 arguments, but received only 2. As we learned earlier in this lesson, arguments are the values in between the ( and ) of function calls. This error even gave us a handy link the circle() reference page! Looking at the reference page we learn that the third missing argument is diameter. By adding a diameter of 40, our code becomes:

function setup() {
    createCanvas(400, 400);
    circle(40,40,40);
  }

And it works! A canvas is made and a circle is drawn without any errors. The lesson here is to not be afraid of errors. They are very useful tools in finding your mistakes and getting your program working.
